Use Cases and Functionalities Selected for First Increment Prototype

For the first increment prototype, three Use Cases have been selected that collectively represent approximately 20% of the overall system functionality. These Use Cases were chosen to establish the system’s foundational features—authentication, real-time analytics visualization, and spatial labeling for behavioral analysis.
The selection was made in consultation with the Project Advisor, and has received explicit approval for inclusion in the First Increment Product to be presented on January 7, 2026.
Selected Use Cases

Figure 1 Selected Use Cases
UC-01: Authenticate Manager
This Use Case ensures that only authorized personnel can access the system’s operations dashboard. Upon successful authentication, the system validates user credentials and grants access to real-time analytics features.
UC-02: View Operations Dashboard 
This Use Case enables store managers to view a real-time operations dashboard displaying key performance metrics derived from in-store customer activity.
It allows managers to monitor visitor counts, demographic breakdowns, and live occupancy heatmaps generated from camera feeds.
UC-08: Zone Labeling
This Use Case enables the manager to manually define or adjust entrance and exit zones on the dashboard interface. These labeled zones assist the analytics engine in accurately categorizing traffic flow and generating behavior-based insights.
This functionality represents a subset of the broader UC-06 “Analyze Customer Activity” Use Case.
Corresponding Functional Requirements
The selected Use Cases directly correspond to the following Functional Requirements from the Software Requirements Specification (SRS):
FReq 1: Manager Authentication and Dashboard Access
Description:
Validate that the user is authenticated and that customer analytics (visitor counts, demographics, heatmaps) are correctly displayed and updated in real time.
Relevant to: UC-01 Authenticate Manager, UC-08 Zone Labeling
FReq 2: Real-Time Dashboard Data Display
Description:
The system shall allow the manager to interactively switch between visualization modes and view AI-generated analytics without leaving the dashboard.
Relevant to: UC-02: View Operations Dashboard
FReq 4: AI-Generated Insights
Description:
The system shall analyze behavioral and occupancy data to produce decision-support insights for management.
Relevant to: UC-08 Zone Labeling
Advisor Approval Statement
The selection of UC-01 (Authenticate Manager), UC-02 (View Operations Dashboard), and UC-08 (Zone Labeling) as the foundation for the Requirements Prototype and First Increment Product has been explicitly approved by the Project Advisor following a review meeting with the development team.

First Increment Prototype

UC-01: Authenticate Manager
Overview:
The Authenticate Manager Use Case represents the system’s initial access control mechanism. It ensures that only authorized users can log into the platform and access the Operations Dashboard. This step forms the security foundation of the ObservAI system, guaranteeing that sensitive customer analytics data is only available to verified managers.
Visual Reference:

Figure 2 Manager Authentication Screen

Functional Requirement Reference:
FReq1: The system shall authenticate store managers through secure credential verification before granting access to the Operations Dashboard.
Use Case: UC1 – Authenticate Manager
Description and Interaction Steps:
System Display:
The system presents the ObservAI Login Screen as in Figure 2 to the user upon startup or when authentication is required.
User Input – Email Address:
The manager enters their registered email (e.g., admin@observai.com) in the “Email address” field.
User Input – Password:
The manager types their password into the “Password” field (The password characters are masked for security reasons).
Optional Step – Remember Me:
The user may check “Remember me for 30 days” to enable persistent login for future sessions.
Authentication Action:
Clicking “Sign in” sends the credentials to the authentication service.
The system validates the input against stored credentials.
If valid, access is granted to the Operations Dashboard (UC2).
If invalid, an error message appears (“Incorrect email or password”).
Alternative Option – Forgot Password:
If the user clicks “Forgot password?”, the system redirects to the password recovery workflow.
Alternative Option – Demo Login:
By clicking “Use Demo Account”, a test account is automatically loaded using the demo credentials displayed at the bottom:
Email: admin@observai.com
Password: demo1234
This allows new users or stakeholders to explore the platform without real credentials.

Sign-up Option:
If the user does not have an account, they can click “Sign up” to initiate the registration process.
Expected Outcome:
Successful login authenticates the manager and transitions the interface to the Operations Dashboard (UC2).
UC-02: View Operations Dashboard
Overview:
 The View Operations Dashboard Use Case allows authenticated store managers to access a real-time analytics interface displaying customer behavior insights derived from in-store camera feeds. The dashboard aggregates live video analytics—visitor counts, demographic summaries, and movement heatmaps—into a unified visualization panel.

Visual Reference:
Figure 3 Gender Distribution Dashboard Screen

Figure 4 Age Distribution Dashboard Screen
Functional Requirement Reference:
FReq2: The system shall allow the manager to interactively switch between visualization modes and view AI-generated analytics without leaving the dashboard.
Use Case: UC-02 – View Operations Dashboard
Description and Interaction Steps:
Access Dashboard:
The Administrator/Analyst logs in successfully (per UC1) and selects the "Analytics Dashboard" from the main navigation.
Gender Distribution Chart:
The system queries the user database for gender data.
A Doughnut Chart renders showing the percentage split between Male (blue segment) and Female (pink segment).
The legend displays the exact percentages (e.g., Male: 65%, Female: 35%).
Hovering over a segment reveals a tooltip with the absolute number of users in that category (e.g., “Male: 65 users”).
Age Distribution Chart:
Below the gender chart, the system displays a Bar Chart for age distribution.
Users are grouped into four brackets:
2-17 (Children/Teens)
18-34 (Young Adults)
35-54 (Middle-Aged Adults)
55+ (Seniors)
Each age group is represented by a blue bar showing the percentage of total users (e.g., 29% for 35-54).
The Y-axis indicates percentage (0% to 30%), and the X-axis lists age groups with icons for clarity.
The highest bar (35-54) highlights the dominant demographic segment.
Data Verification:
The administrator can verify data completeness by confirming the sum of age group percentages equals 100%.
Preconditions:
User is authenticated as Administrator or Analyst with dashboard access.
Postconditions:
Demographic charts are rendered and displayed correctly on the screen.
Expected Outcome:
The administrator views a clear, interactive visual summary of user demographics, aiding data-driven decision making.
UC-08: Zone Labeling
Overview:
The Label Entrance/Exit Zones Use Case allows store managers to manually define, edit, or confirm the physical entry and exit regions within the store layout displayed on the Operations Dashboard.
Accurate labeling of these zones improves the AI analytics engine’s ability to categorize customer flow, measure occupancy by zone, and generate behavior-based insights such as queue density and peak movement periods.
Visual Reference:

Figure 5 Zone Labeling Interface on Operations Dashboard

Functional Requirement Reference:
FReq1: Manager Authentication and Dashboard Access – Display real-time store layout and analytics.
FReq4: AI-Generated Insights – Improve accuracy of customer flow and occupancy data through accurate zone labeling.
Use Case: UC8 – Zone Labeling
Description and Interaction Steps:
Access Zone Labeling Interface:
The Store Manager logs into the Operations Dashboard and selects the “Label Zones” feature from the menu.
Display Store Layout:
The system presents the store floor plan with existing or default entrance and exit zones highlighted by translucent rectangles.
Define or Edit Zones:
The manager can click and drag on the floor plan to create new rectangular zones representing physical entrances or exits.
Existing zones can be resized, repositioned, or deleted by direct manipulation.
Confirm Zones:
After labeling, the manager confirms the entrance and exit zones by clicking “Save” or “Confirm.”
The system validates that zones do not overlap incorrectly or fall outside the floor plan boundaries.
Integration with AI Analytics:
Accurately labeled zones improve the AI analytics engine’s ability to track customer flow, occupancy by zone, queue density, and peak movement times.
Feedback and Updates:
The system provides immediate visual feedback on the zones and displays a success message confirming the update.
Labeled zones are stored and used in real-time customer monitoring and reporting.
Preconditions:
The manager must be logged into the Operations Dashboard with permissions to label zones.
Postconditions:
Entrance and exit zones are accurately defined and saved for use by AI analytics and live monitoring.
Expected Outcome:
The manager successfully defines or updates physical entrance and exit zones on the store layout, enabling more accurate real-time analytics and operational insights.
